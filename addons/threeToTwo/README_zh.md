# threeToTwo 插件 - Godot三渲二工具

## 概述

threeToTwo 是一个 Godot 引擎插件，专门用于将 3D 模型渲染为 2D 贴图（三渲二）。该插件采用全新的架构设计，深度集成到 Godot 编辑器中，充分利用编辑器的原生功能，提供高效、灵活的 3D 转 2D 工作流程。

### 设计理念

与传统的独立窗口式插件不同，threeToTwo 采用 **编辑器集成** 设计理念：
- **利用 Godot 原生功能**：模型移动、旋转、缩放、灯光设置等操作直接在 Godot 编辑器中进行
- **右侧停靠面板**：作为编辑器的扩展，提供专门的三渲二功能
- **实时同步**：与编辑器场景实时交互，所见即所得
- **多模型管理**：支持同时管理多个模型和动画

### 主要功能

- **3D 模型导入** - 支持 FBX、GLTF、GLB、OBJ 等主流 3D 格式，自动处理导入流程
- **相机管理** - 选择场景中的相机，实时同步相机变换，支持场景相机跟随
- **AnimationPlayer 管理器** - 批量添加、控制多个 AnimationPlayer，统一管理动画播放
- **实时预览** - 在插件面板中实时查看 3D 模型渲染效果
- **材质切换** - 在普通材质和法线材质之间切换，满足不同渲染需求
- **特效纹理缓存** - 捕获动画序列，生成特效纹理用于 VFX 制作
- **贴图导出** - 将 3D 模型渲染为 2D 贴图导出，支持单帧和批量导出
- **相机配置** - 保存和加载相机配置文件，快速恢复工作状态

## 安装指南

### 方法一：直接复制（推荐）

1. 将 `addons/threeToTwo` 文件夹复制到您的 Godot 项目的 `addons` 目录下
2. 如果项目中没有 `addons` 目录，请先创建该目录
3. 打开 Godot 编辑器，进入 **项目设置（Project Settings）**
4. 选择 **插件（Plugins）** 选项卡
5. 找到 **threeToTwo** 插件并启用它

### 方法二：通过 Git 子模块

```bash
# 在您的 Godot 项目目录中
git submodule add https://github.com/wuyuanlch/3DTo2D.git addons/threeToTwo
```

启用插件后，Godot 编辑器右侧会出现 **3Dto2D** 停靠面板。

## 快速开始

### 第一步：设置场景

1. 在 Godot 编辑器中创建或打开一个 3D 场景
2. 添加 Camera3D 节点作为渲染相机
3. 添加 Node3D 节点作为模型容器（可选）
4. 添加 AnimationPlayer 节点并设置动画（可选）

### 第二步：使用插件面板

1. 启用插件后，右侧停靠面板会出现 **3Dto2D** 面板
2. 点击 **"获取选中相机"** 按钮选择场景中的 Camera3D 节点
3. 点击 **"获取选中 ModelContainer"** 按钮选择模型容器节点
4. 点击 **"获取选中 AnimationPlayer"** 按钮添加 AnimationPlayer 到管理器

### 第三步：预览和导出

1. 点击 **"图片和动画导出"** 按钮打开预览面板
2. 在预览面板中调整贴图尺寸和材质
3. 点击相应的导出按钮导出贴图

## 界面详解

### 右侧停靠面板布局

插件主界面位于 Godot 编辑器右侧停靠面板，分为以下几个主要区域：

#### 1. 相机管理区域
- **相机标签**：显示当前选中的相机节点名称
- **"获取选中相机"按钮**：选择场景中的 Camera3D 节点
- **实时同步复选框**：启用/禁用相机变换实时同步
- **场景跟随复选框**：启用/禁用场景相机跟随模式

#### 2. 模型容器区域
- **模型容器标签**：显示当前选中的模型容器节点
- **"获取选中 ModelContainer"按钮**：选择场景中的 Node3D 节点作为模型容器
- **"上传模型到容器"按钮**：上传 3D 模型文件到选中的容器

#### 3. AnimationPlayer 管理器区域
- **"获取选中AnimationPlayer"按钮**：添加选中的 AnimationPlayer 到管理器
- **AnimationPlayer 列表**：显示已添加的所有 AnimationPlayer
- **控制按钮组**：
  - **"播放所有"**：播放所有选中的 AnimationPlayer
  - **"暂停所有"**：暂停所有选中的 AnimationPlayer  
  - **"停止所有"**：停止所有选中的 AnimationPlayer

#### 4. 功能按钮区域
- **"图片和动画导出"按钮**：打开预览面板
- **"缓存特效"按钮**：开始/停止特效纹理缓存
- **"加载相机配置"按钮**：加载相机配置文件

### AnimationPlayer 条目

每个添加到管理器的 AnimationPlayer 都会显示一个条目，包含：
- **动画名称标签**：显示 AnimationPlayer 节点名称
- **动画选择下拉菜单**：选择要播放的动画
- **播放/暂停按钮**：控制该 AnimationPlayer 的播放状态
- **选择复选框**：启用/禁用该 AnimationPlayer 参与全局控制
- **循环复选框**：启用/禁用动画循环播放
- **删除按钮**：从管理器中移除该 AnimationPlayer

### 预览面板布局

点击 **"图片和动画导出"** 按钮打开预览面板，提供贴图预览和导出功能：

#### 1. 预览区域
- **贴图显示**：显示当前渲染的 2D 贴图
- **缩放控制**：支持鼠标滚轮缩放和拖动查看

#### 2. 控制工具栏
- **关闭按钮**：关闭预览面板
- **缩放按钮**：放大、缩小、重置缩放
- **缩放百分比输入框**：精确控制缩放比例

#### 3. 尺寸控制
- **宽度输入框**：设置贴图宽度（像素）
- **高度输入框**：设置贴图高度（像素）

#### 4. 材质控制
- **材质选择器**：在普通材质和法线材质之间切换
- **导出按钮组**：
  - **导出当前贴图**：导出当前显示的贴图
  - **导出普通贴图**：导出普通材质贴图
  - **导出法线贴图**：导出法线材质贴图
  - **导出两种材质**：同时导出普通和法线贴图

#### 5. 动画控制
- **时间滑块**：拖动调整动画进度
- **播放/暂停按钮**：控制动画播放
- **上一帧/下一帧按钮**：逐帧控制动画
- **帧信息标签**：显示当前时间/总时长

#### 6. 批量导出
- **间隔时间输入框**：设置批量导出的时间间隔（秒）
- **批量导出按钮**：开始批量导出动画帧

#### 7. 插值算法
- **算法选择器**：选择图像缩放插值算法
  - **Nearest**：最近邻插值（像素化效果）
  - **Bilinear**：双线性插值（平滑效果）
  - **Cubic**：三次插值（更平滑）
  - **Lanczos**：Lanczos 插值（高质量）

### VFX 预览面板布局

点击 **"缓存特效"** 按钮后，会弹出 VFX 预览面板：

#### 1. 预览区域
- **纹理序列显示**：显示缓存的纹理序列
- **播放控制**：播放/暂停纹理序列

#### 2. 材质参数控制
- **阈值滑块**：调整材质阈值（0.0-1.0）
- **柔化滑块**：调整材质边缘柔化（0.0-0.5）
- **背景颜色选择器**：选择背景颜色

#### 3. 导出控制
- **导出按钮**：导出当前显示的纹理

## 功能使用详解

### 1. 相机管理

#### 选择相机
1. 在场景中选择一个 Camera3D 节点
2. 点击插件面板中的 **"获取选中相机"** 按钮
3. 相机名称会显示在相机标签中

#### 实时同步
- 启用 **实时同步复选框** 后，选中的相机变换会实时同步到预览视口
- 在 Godot 编辑器中移动、旋转相机时，预览会实时更新
- 禁用后，相机变换不会自动同步

#### 场景相机跟随
- 启用 **场景跟随复选框** 后，选中的相机会跟随编辑器场景相机
- 在 3D 编辑器中移动视角时，选中的相机也会同步移动
- 适合需要从编辑器视角预览渲染效果的场景

### 2. 模型容器管理

#### 选择模型容器
1. 在场景中选择一个 Node3D 节点（或任何 3D 节点）
2. 点击插件面板中的 **"获取选中 ModelContainer"** 按钮
3. 容器名称会显示在模型容器标签中

#### 上传模型
1. 先选择一个模型容器节点
2. 点击 **"上传模型到容器"** 按钮
3. 在文件对话框中选择 3D 模型文件（支持 FBX、GLTF、GLB、OBJ）
4. 插件会自动处理导入流程：
   - 检查模型是否已导入
   - 如果未导入，会自动复制文件到项目目录
   - 等待 Godot 完成模型导入
   - 加载模型到选中的容器

### 3. AnimationPlayer 管理器

#### 添加 AnimationPlayer
1. 在场景中选择一个 AnimationPlayer 节点
2. 点击 **"获取选中AnimationPlayer"** 按钮
3. AnimationPlayer 会添加到管理器中，显示为条目

#### 管理多个动画
- 可以添加多个 AnimationPlayer 到管理器
- 每个 AnimationPlayer 可以独立选择动画、控制播放
- 使用 **选择复选框** 控制哪些 AnimationPlayer 参与全局控制

#### 全局控制
- **"播放所有"**：播放所有选中的 AnimationPlayer
- **"暂停所有"**：暂停所有选中的 AnimationPlayer
- **"停止所有"**：停止所有选中的 AnimationPlayer

#### 动画同步
- 所有选中的 AnimationPlayer 会同步播放
- 在预览面板中控制时间时，所有动画会同步到相同时间点
- 支持循环和非循环模式

### 4. 实时预览

#### 打开预览
1. 设置好相机和模型后，点击 **"图片和动画导出"** 按钮
2. 预览面板会显示当前相机视角下的渲染结果
3. 预览图像会根据设置的尺寸自动调整

#### 预览交互
- **鼠标滚轮**：缩放预览图像
- **鼠标中键拖动**：在放大状态下拖动查看图像不同部分
- **缩放按钮**：放大、缩小、重置缩放比例
- **缩放输入框**：输入百分比精确控制缩放

### 5. 材质切换

#### 普通材质
- 显示模型的原始材质效果
- 适合大多数导出需求
- 保持模型的纹理和颜色信息

#### 法线材质
- 使用特殊的法线着色器渲染
- 显示模型的法线信息
- 输出 RGB 颜色表示法线方向
- 用于特殊渲染效果或后期处理

#### 切换效果
- 切换材质后，预览会立即更新
- 所有模型网格会同时应用新材质
- 法线材质模式下会显示紫色背景的法线网格

### 6. 贴图导出

#### 单张导出
1. 在预览面板中调整好贴图尺寸和材质
2. 点击相应的导出按钮：
   - **导出当前贴图**：导出当前显示的贴图
   - **导出普通贴图**：切换到普通材质并导出
   - **导出法线贴图**：切换到法线材质并导出
   - **导出两种材质**：同时导出普通和法线贴图
3. 在文件对话框中选择保存位置和文件名
4. 贴图将以 PNG 格式保存

#### 导出设置
- **文件名自动生成**：基于动画名和导出序号自动生成文件名
- **格式固定**：始终导出为 PNG 格式，支持透明通道
- **尺寸保持**：保持预览时设置的尺寸

### 7. 动画控制

#### 播放控制
1. 在预览面板中，使用时间滑块或输入框设置时间
2. 点击 **播放** 按钮开始播放动画
3. 点击 **暂停** 按钮暂停播放
4. 使用 **上一帧/下一帧** 按钮逐帧控制

#### 时间同步
- 所有选中的 AnimationPlayer 会同步到相同时间点
- 支持循环播放和非循环播放
- 时间滑块显示当前时间/总时长

### 8. 批量导出

#### 批量导出流程
1. 在预览面板的 **批量导出** 区域设置时间间隔（秒）
2. 点击 **"批量导出"** 按钮
3. 在文件对话框中选择保存目录和文件名前缀
4. 插件会自动：
   - 创建以文件名前缀命名的目录
   - 在目录下创建 "普通贴图" 和 "法线贴图" 子目录
   - 按时间间隔导出动画的每一帧
   - 分别保存普通材质和法线材质的贴图

#### 批量导出设置
- **时间间隔**：导出帧之间的时间间隔（秒）
  - 例如：设置为 0.5 表示每 0.5 秒导出一帧
  - 设置为 1.0 表示每秒导出一帧
- **导出范围**：从动画开始到结束的所有帧
- **文件命名**：`前缀_序号.png` 和 `前缀_Normal_序号.png`

### 9. 特效纹理缓存

#### 缓存流程
1. 在插件面板中点击 **"缓存特效"** 按钮
2. 插件会自动：
   - 播放所有选中的动画
   - 按固定间隔（0.01秒）捕获纹理
   - 缓存整个动画序列的纹理
3. 缓存完成后，会自动弹出 VFX 预览面板

#### VFX 预览面板
- 显示缓存的纹理序列
- 支持播放/暂停纹理序列
- 可以调整材质参数（阈值、柔化、背景颜色）
- 可以导出当前显示的纹理

#### 应用场景
- 制作特效序列图
- 生成粒子系统纹理
- 创建动态贴图资源

### 10. 相机配置文件

#### 保存相机配置
相机配置可以通过代码保存，包含：
- 相机位置、旋转、缩放
- 相机属性（FOV、近平面、远平面）
- 导出尺寸设置

#### 加载相机配置
1. 点击 **"加载相机配置"** 按钮
2. 选择相机配置文件（JSON 格式）
3. 插件会自动：
   - 应用相机位置和旋转
   - 更新预览面板的尺寸设置
   - 恢复工作状态

#### 配置文件格式
```json
{
  "version": "1.0",
  "camera": {
	"position": {"x": 0, "y": 0, "z": 5},
	"rotation": {"x": 0, "y": 0, "z": 0, "w": 1},
	"fov": 70,
	"near": 0.1,
	"far": 100,
	"scale": {"x": 1, "y": 1, "z": 1}
  },
  "export_size": {
	"width": 512,
	"height": 512
  }
}
```

### 11. 插值算法选择

#### 可用的插值算法
1. **Nearest（最近邻）**
   - 效果：像素化，保持硬边缘
   - 适用：像素艺术、复古风格
   - 性能：最快

2. **Bilinear（双线性）**
   - 效果：平滑，中等质量
   - 适用：一般用途，平衡质量和性能
   - 性能：较快

3. **Cubic（三次）**
   - 效果：更平滑，高质量
   - 适用：需要高质量缩放的场景
   - 性能：较慢

4. **Lanczos**
   - 效果：最高质量，锐利清晰
   - 适用：专业用途，需要最佳质量
   - 性能：最慢

#### 选择建议
- 小尺寸贴图放大：使用 **Lanczos** 或 **Cubic** 保持清晰度
- 大尺寸贴图缩小：使用 **Bilinear** 平衡质量和性能
- 像素艺术风格：使用 **Nearest** 保持像素感
- 一般游戏资源：使用 **Bilinear** 或 **Cubic**

## 高级功能

### 1. 多 AnimationPlayer 同步

#### 同步原理
- 所有选中的 AnimationPlayer 使用相同的全局时间线
- 每个 AnimationPlayer 播放自己的选中动画
- 支持不同长度的动画同步播放

#### 循环处理
- 如果动画设置为循环，播放到结尾后会重新开始
- 非循环动画播放到结尾后会暂停
- 所有动画的循环设置独立控制

### 2. 实时同步机制

#### 同步流程
1. 插件在 `_process` 函数中检查相机变换
2. 如果启用实时同步，将相机变换应用到预览视口相机
3. 强制更新 SubViewport 渲染

#### 性能优化
- 只在变换发生变化时更新
- 使用缓存减少不必要的计算
- 支持禁用实时同步以提升性能

### 3. 模型导入系统

#### 自动导入流程
1. 检查模型是否已导入（检查 .import 文件）
2. 如果未导入，复制文件到项目目录
3. 等待 Godot 完成模型导入
4. 加载模型到选中的容器

#### 导入优化
- **智能检测**：自动检测模型是否已导入，避免重复导入
- **异步处理**：导入过程不会阻塞编辑器界面
- **错误处理**：提供详细的错误信息和恢复建议

### 4. 纹理缓存系统

#### 缓存机制
1. 按固定时间间隔（0.01秒）捕获纹理
2. 将纹理存储在内存数组中
3. 支持播放缓存的纹理序列

#### 性能考虑
- 缓存纹理会占用内存，建议及时清理
- 支持中途停止缓存
- 缓存完成后自动弹出预览面板

## 常见问题

### Q1: 插件面板没有显示怎么办？
**A:** 请检查以下事项：
1. 插件是否已正确启用（项目设置 → 插件）
2. 是否重启了 Godot 编辑器
3. 查看控制台是否有错误信息
4. 确保 Godot 版本兼容（建议 Godot 4.4+）

### Q2: 无法选择相机或模型容器？
**A:** 请确认：
1. 在场景中选中了正确的节点类型
   - 相机：必须是 Camera3D 节点
   - 模型容器：可以是任何 Node3D 节点
2. 节点在场景树中且有效
3. 尝试重新选择节点

### Q3: 预览面板显示空白或错误？
**A:** 可能的原因：
1. 没有选中有效的相机节点
2. 相机没有正确设置
3. 模型不在相机视野内
4. 实时同步被禁用

### Q4: 导出的贴图尺寸不正确？
**A:** 请检查：
1. 预览面板中的宽度和高度设置
2. 是否在导出前修改了尺寸但未确认
3. 尝试点击输入框外部或按回车键确认尺寸

### Q5: 法线材质显示为紫色？
**A:** 这是正常现象：
1. 法线材质使用特殊的着色器渲染法线信息
2. 紫色背景是法线着色器的默认颜色
3. 当模型有法线信息时，会显示为彩色法线贴图
4. 如果模型没有法线信息，会保持紫色背景

### Q6: 动画不同步或播放异常？
**A:** 请检查：
1. 所有 AnimationPlayer 是否都已正确添加到管理器
2. 是否选中了参与全局控制的 AnimationPlayer
3. 动画是否设置为循环播放
4. 时间滑块是否在有效范围内

### Q7: 批量导出时卡住或停止？
**A:** 可能的原因和解决方法：
1. **动画太长**：减少时间间隔或导出范围
2. **贴图尺寸太大**：减小宽度和高度设置
3. **磁盘空间不足**：检查保存目录的可用空间
4. **文件权限问题**：确保有写入权限
5. **尝试重新启动插件**：关闭并重新打开插件窗口

### Q8: 特效纹理缓存失败？
**A:** 请确认：
1. 是否有选中的 AnimationPlayer
2. 动画是否能够正常播放
3. 内存是否充足
4. 尝试减少缓存时间间隔
